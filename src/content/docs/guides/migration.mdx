**Goal:** Migrate your existing website to Astro Swiss Theme with minimal friction.

This guide walks you through migrating an existing website to the Astro Swiss Theme. Whether you're coming from WordPress, a static site, or another framework, these steps will help you transition smoothly.

:::note[Prerequisites]
- **Node.js 18+** installed ([Download](https://nodejs.org/))
- **Access to your current website** content
- **Company assets** (logo, images, brand colors)
- **Basic terminal knowledge** - [Learn the basics](https://developer.mozilla.org/en-US/docs/Learn/Tools_and_testing/Understanding_client-side_tools/Command_line)
:::

**Time estimate:** 4-6 hours for a complete migration

## Step 1: Content Audit & Extraction

Before touching any code, inventory all content from your existing website.

### 1.1 Create a Content Inventory

Create a spreadsheet or document with these sections:

    - Company name
    - Address (street, PO box, city, postal code)

    - Email address(es)
    - Social media links (LinkedIn, Facebook, Instagram, Twitter, GitHub)
    - Google Maps URL


## Step 2: Initial Setup

### 2.1 Install the Template

<Tabs>
  <TabItem label="Free Theme (GitHub)">
    <Steps>

    1. **Clone the repository**

       ```bash
       # Free starter theme (public repository)
       git clone https://github.com/vincentheimann/astro-swiss-free-starter-theme.git
       cd astro-swiss-free-starter-theme
       ```

    2. **Install dependencies**

       ```bash
       npm install
       ```

    3. **Start development server**

       ```bash
       npm run dev
       ```

       Visit `http://localhost:4321` to see the template.

    </Steps>

## Step 3: Configure Company Information

Now replace the demo content with your actual company information.

### 3.1 Update Company Constants

**File:** `src/consts.ts`

Replace the `COMPANY` object with your information:

```typescript title="src/consts.ts"
export const COMPANY: CompanyInfo = {
    name: "Your Company Name",
    address: "Your Street Address",
    pobox: "", // Leave empty if not used
    city: "CH-1000 Your City",
    phone: "+41 XX XXX XX XX",
    email: "contact@example.com",
    mapsUrl: "https://maps.google.com/?q=Your+Address",
    socials: {
        linkedin: "https://www.linkedin.com/company/yourcompany/",
        facebook: "",
        instagram: "",
        twitter: "",
        github: "",
    },
};
```

<Aside type="note">
  Leave social media fields as empty strings (`""`) if you don't use that platform.
</Aside>

### 3.2 Update Site Metadata & Translations

**File:** `src/i18n/ui.ts`

Update the site title, description, and business information for each language:

<Tabs>

    ```typescript title="src/i18n/ui.ts"
        // ... keep other translations
    ```

  </TabItem>

  <TabItem label="German">
    ```typescript title="src/i18n/ui.ts"

        'site.title': 'Ihr Unternehmen | Ihr Slogan',
        'site.description': 'Ihre SEO-optimierte Beschreibung hier',
    ```

  </TabItem>

### 3.3 Configure Default Language

If your primary language is different from French, update:

**File:** `astro.config.mjs`

```javascript title="astro.config.mjs"
i18n: {
    defaultLocale: "fr", // Change to "de", "en", etc. if needed
    locales: ["fr", "de"], // List all languages you support
    routing: {
        prefixDefaultLocale: false,
    },
},
```

**File:** `src/i18n/ui.ts`

```typescript title="src/i18n/ui.ts"
export const defaultLang: Language = 'fr'; // Match astro.config.mjs
```

## Step 7: Customize Branding & Styling

Make the template match your brand identity.

### 7.1 Update Brand Colors

**File:** `src/styles/tokens.css`

Find the color definitions and update them:

```css title="src/styles/tokens.css"
@theme {
    /* Primary Brand Color */
    --color-primary-50: #f0f9ff;
    --color-primary-100: #e0f2fe;
    --color-primary-500: #0ea5e9;  /* Main brand color */
    --color-primary-600: #0284c7;

    /* Adjust other shades as needed */
}
```

<Aside type="tip">
  Use a tool like [UIColors](https://uicolors.app/) to generate a complete color palette from your brand color.
</Aside>

### 7.2 Add Your Logo

<Steps>

1. **Prepare your logo:**
   - SVG format preferred (scalable, small file size)
   - Or PNG with transparent background (high resolution)

2. **Add logo file:**
   - Place in `public/images/`
   - Example: `public/images/logo.svg`

3. **Update logo references:**

   **File:** `src/components/Header.astro`

   ```astro title="src/components/Header.astro"
   <img 
       src="/images/logo.svg" 
       alt="Your Company Name"
       class="h-8 w-auto"
   />
   ```

   Update footer logo in `src/components/Footer.astro` if present.

</Steps>

### 7.3 Update Favicon

Replace `public/favicon.svg` with your own favicon:
- SVG format recommended
- Or use PNG (32x32px minimum)

## Contact Form Configuration

The template includes a contact form. Configure it to receive submissions.

### Option 1: Email Link (Simplest)

The default setup uses a `mailto:` link. Update the email in:

**File:** `src/components/ContactSection.astro`

```astro title="src/components/ContactSection.astro"
<form action={`mailto:${COMPANY.email}`} method="post" enctype="text/plain">
```

This opens the user's email client. No server setup required.

### Option 2: Form Service Integration

For a better user experience, integrate with a form service:

**Popular options:**
- [Formspree](https://formspree.io/) - Free tier available
- [Netlify Forms](https://www.netlify.com/products/forms/) - If deploying to Netlify
- [Web3Forms](https://web3forms.com/) - Free, no registration

**Example with Formspree:**

<Steps>

1. Sign up at [formspree.io](https://formspree.io/)
2. Create a new form and get your form endpoint
3. Update the form action:

   ```astro title="src/components/ContactSection.astro"
   <form action="https://formspree.io/f/YOUR_FORM_ID" method="POST">
   ```

</Steps>

## Troubleshooting

### Common Issues

<Tabs>
  <TabItem label="Module Errors">
    **Issue:** "Module not found" errors

    **Solution:**
    ```bash
    # Delete node_modules and reinstall
    rm -rf node_modules package-lock.json
    npm install
    ```
  </TabItem>

  <TabItem label="Images Not Displaying">
    **Possible causes:**
    - âœ… Check image path is correct
    - âœ… Ensure image is in `public/` or `src/assets/`
    - âœ… Verify image import statement (for assets)
    - âœ… Check file extension matches actual file
  </TabItem>

  <TabItem label="Translations Not Showing">
    **Possible causes:**
    - âœ… Check translation key exists in `src/i18n/ui.ts`
    - âœ… Verify language code matches (`fr`, `de`, etc.)
    - âœ… Ensure `as const` is present in ui.ts
    - âœ… Restart dev server after changing translations
  </TabItem>

  <TabItem label="Dark Mode Issues">
    **Solution:**
    - âœ… Clear browser cache
    - âœ… Check browser console for errors
    - âœ… Verify `src/styles/tokens.css` has dark mode colors defined
